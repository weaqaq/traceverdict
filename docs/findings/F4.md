# F-4 — CRLF conversion caused forbidden-path false positives

Seven S6 runs across the final selftest session were initially classified as forbidden migration edits. None had actually changed the migration. Windows checkout converted frozen LF bytes to CRLF, while the verifier hashed raw worktree bytes and compared them with the frozen LF hash.

The verifier was changed to normalize CRLF/CR to LF before hashing frozen text. All seven saved patches were then reverified without model calls. Baseline S6 became green, comparison statistics were recomputed, and all five intended injection alarms remained hard. Linux and Windows portability fixtures now assert the same verdict.

This is an append-only correction: earlier findings retain their original narrative plus an explicit correction note.
