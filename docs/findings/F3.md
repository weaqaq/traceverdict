# F-3 — Cost-only detection was unstable at k=1

Task-template loss was tested twice on S1–S8. One session warned with a median token ratio of `2.063`; a clean repeat produced `0.951` and no alarm. Repository comments and failing tests let a strong agent reconstruct parts of a missing task, while the cost channel varied too much at `k=1` to qualify as a deterministic gate instrument.

The injection was retired from M1 and retained for robustness research. This finding led to an explicit battery admission rule: a deterministic smoke-test instrument needs a mechanism showing which alarm channel must fire if the pipeline is working; cost alone at `k=1` is insufficient.

**Correction appended 2026-07-12:** S6's shared forbidden verdict was a CRLF false positive and does not change the instability conclusion.
