# F-1 — Removing tool instructions did not degrade the tested agent

On 2026-07-11, an injection removed bash/editing instructions while preserving the task and submission protocol. Across self tasks S1–S8 at `k=1`, baseline and candidate both passed 7/8: delta pass `0`, bootstrap CI `[0,0]`, exact McNemar `p=1.0`, alarm `none`.

The injection was directly confirmed in the rendered prompt, and the median token ratio fell to `0.547`. The honest conclusion is narrow: for mini-swe-agent 2.4.5 with the tested thinking model, this change was not a known-bad regression at the available resolution. It left the M1 detector denominator and remains a possible M3 robustness arm.

**Correction appended 2026-07-12:** the shared S6 failure was a CRLF verifier false positive, not a migration edit. Reverification changed S6 to pass in both arms, leaving delta pass and this finding unchanged.
